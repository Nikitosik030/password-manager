# Руководство пользователя

## password-manager — безопасное хранение паролей

Приложение для безопасного хранения паролей с шифрованием AES-128, генерацией и оценкой стойкости. Доступно в двух версиях: **GUI** (графический интерфейс) и **CLI** (консольный интерфейс).

---

## Запуск

### Локально (GUI)

```bash
git clone https://github.com/Nikitosik030/password-manager.git
cd password-manager
pip install -r requirements.txt
python -m src.main
```

### Локально (CLI)

```bash
python -m src.main init
python -m src.main add --login test --generate
python -m src.main list
```

### Через Docker

```bash
docker build -t password-manager .
docker run -it --rm -v $(pwd)/data:/app/data password-manager init
```

### EXE-файл (Windows)

Скачайте `PasswordManager.exe` из папки `dist/` и запустите двойным кликом.

## Интерфейс пользователя (GUI)

### Главное меню (GUI)

После запуска GUI отображается экран входа:

```

Путь к хранилищу: [data/vault.vault]
```

* **«Войти»** — открыть существующее хранилище.
* **«Создать»** — создать новое хранилище с указанным мастер-паролем.

### Управление в GUI

| Действие | Способ |
| :--- | :--- |
| **Добавить запись** | Кнопка «➕ Добавить» |
| **Удалить запись** | Выбрать запись в таблице → Кнопка «🗑️ Удалить» |
| **Просмотреть запись** | Двойной клик по нужной записи |
| **Копировать пароль** | В окне просмотра записи → Кнопка «📋 Копировать пароль» |
| **Поиск записей** | Ввести текст в поле «Поиск» и нажать `Enter` |
| **Экспорт хранилища** | Кнопка «💾 Экспорт» → Выбрать место на диске |

### 1. Добавление записи (GUI)

1. Нажмите кнопку **«➕ Добавить»**.
2. Заполните поля:
   * **Логин** — имя пользователя (обязательное поле).
   * **Пароль** — можно ввести вручную или сгенерировать автоматически.
   * **URL** — адрес веб-сайта (опционально).
   * **Заметка** — дополнительная текстовая информация (опционально).
3. Нажмите кнопку **«Сохранить»**.

#### Генерация пароля в GUI
1. В окне добавления записи нажмите кнопку **«Сгенерировать»**.
2. Пароль автоматически подставится в соответствующее поле.
3. При сохранении записи в системе будет наглядно показана его стойкость.

## Консольный интерфейс (CLI)

### 2. Добавление записи (CLI)

```bash
# С автоматической генерацией пароля
python -m src.main add --login myemail@mail.com --generate

# С указанием своего собственного пароля
python -m src.main add --login admin --password MySuperPass123

# С указанием URL и заметки
python -m src.main add --login admin --password MyPass123 --url https://admin.local --note "Админка"

# Полный пример использования всех доступных параметров
python -m src.main add --login user@example.com --password MyPass123! --url https://example.com --note "Рабочий аккаунт"
```

#### Параметры команды `add`:

| Параметр | Описание | Обязательный |
| :--- | :--- | :--- |
| `--login` | Логин пользователя | ✅ Да |
| `--password` | Пароль (если параметр не указан — система запросит его ввод) | ❌ Нет |
| `--generate` | Сгенерировать сложный пароль в автоматическом режиме | ❌ Нет |
| `--url` | Адрес целевого сайта | ❌ Нет |
| `--note` | Произвольная текстовая заметка к записи | ❌ Нет |

### 3. Генератор паролей (CLI)

```bash
# Генерация пароля по умолчанию (длиной 16 символов)
python -m src.main generate

# Генерация пароля указанной длины
python -m src.main generate --length 20

# Генерация длинного пароля (32 символа)
python -m src.main generate --length 32
```

### 4. Проверка стойкости пароля (CLI)

```bash
# Проверить стойкость конкретного пароля
python -m src.main check "MyPassword123!"

# Проверить стойкость автоматически сгенерированного пароля
python -m src.main check "$(python -m src.main generate --length 20)"
```

### 5. Поиск записей (CLI)

```bash
# Поиск совпадений по логину, URL или тексту заметки
python -m src.main search "mail"

# Поиск записей по части слова или подстроке
python -m src.main search "admin"
```

### 6. Список всех записей (CLI)

```bash
python -m src.main list
```

### 7. Удаление записи (CLI)

```bash
# Шаг 1: Найти уникальный ID нужной записи через список
python -m src.main list

# Шаг 2: Удалить запись, передав её ID в команду delete
python -m src.main delete 12345678-1234-1234-1234-123456789abc
```

### 8. Экспорт хранилища (CLI)

```bash
# Экспорт в файл по умолчанию (в директорию data/export.vault)
python -m src.main export

# Экспорт с указанием кастомного пути для сохранения файла
python -m src.main export --output /home/user/backup.vault

# Экспорт данных прямо на рабочий стол для операционной системы Windows
python -m src.main export --output C:\Users\username\Desktop\backup.vault
```

### 9. Полный список CLI-команд

| Команда | Описание | Пример использования |
| :--- | :--- | :--- |
| `init` | Создать и инициализировать новое зашифрованное хранилище | `python -m src.main init` |
| `add` | Добавить новую учетную запись в менеджер | `python -m src.main add --login test --generate` |
| `list` | Вывести интерактивный список всех существующих записей | `python -m src.main list` |
| `search` | Выполнить полнотекстовый поиск по ключевому слову | `python -m src.main search "текст"` |
| `delete` | Полностью удалить запись из базы данных по её ID | `python -m src.main delete ID` |
| `export` | Экспортировать текущую базу данных в указанный файл | `python -m src.main export --output file.vault` |
| `generate` | Запустить генератор случайных устойчивых паролей | `python -m src.main generate --length 20` |
| `check` | Выполнить комплексную проверку пароля на стойкость | `python -m src.main check "пароль"` |
| `--help` | Показать справочную информацию по доступным аргументам | `python -m src.main --help` |

## Docker-команды (полный список)

Если пользователь работает на Windows PowerShell, нужно заменить $(pwd) на ${pwd}, если на Windows CMD, то заменить $(pwd) на %cd%. Для Linux и Mac меня не нужно.

```bash
# Сборка Docker-образа приложения
docker build -t password-manager .

# Проверка списка всех доступных CLI-команд внутри контейнера
docker run --rm password-manager --help

# Инициализация нового зашифрованного хранилища
docker run -it --rm -v $(pwd)/data:/app/data password-manager init

# Добавление записи с автоматической генерацией пароля
docker run -it --rm -v $(pwd)/data:/app/data password-manager add --login docker_test --generate

# Добавление записи со своим собственным паролем
docker run -it --rm -v $(pwd)/data:/app/data password-manager add --login admin --password MyPass123

# Добавление записи с одновременным указанием URL и заметки
docker run -it --rm -v $(pwd)/data:/app/data password-manager add --login user@example.com --password Pass123 --url https://example.com --note "Тест"

# Просмотр списка всех сохраненных записей
docker run -it --rm -v $(pwd)/data:/app/data password-manager list

# Поиск записей по совпадению ключевого слова
docker run -it --rm -v $(pwd)/data:/app/data password-manager search docker_test

# Генерация пароля (без необходимости монтирования локальной папки data)
docker run --rm password-manager generate --length 20

# Проверка стойкости пароля (без необходимости монтирования локальной папки data)
docker run --rm password-manager check "MyPassword123!"

# Экспорт хранилища с монтированием папки vault для сохранения бэкапа
docker run -it --rm -v $(pwd)/data:/app/data -v $(pwd)/vault:/app/vault password-manager export --output /app/vault/backup.vault

# Экспорт данных в конкретный файл в текущей рабочей директории
docker run -it --rm -v $(pwd)/data:/app/data -v $(pwd):/app/out password-manager export --output /app/out/my_backup.vault

#Экспорт данных в конктретый файл на пк(заменить "...")
docker run -it --rm -v $(pwd)/data:/app/data -v "путь места для сохранения(Например: C:\Users\nikita\Desktop":/app/backup password-manager export --output /app/backup/vault_backup.vault

# Удаление записи по её идентификатору (замените ID на реальное значение)
docker run -it --rm -v $(pwd)/data:/app/data password-manager delete a1b2c3d4-...
```

## Возможные ошибки и их значение

| Сообщение об ошибке | Причина возникновения |
| :--- | :--- |
| `[!] Введите мастер-пароль` | Обязательное поле ввода мастер-пароля осталось пустым. |
| `[!] Мастер-пароль должен быть минимум 8 symbols` | Длина введенного пароля меньше допустимого лимита безопасности. |
| `[!] Пароли не совпадают` | При инициализации хранилища повторный пароль отличается от первого. |
| `[!] Неверный мастер-пароль` | Указан неправильный пароль при попытке авторизоваться в системе. |
| `[!] Файл не найден` | Файл зашифрованной базы данных отсутствует по указанному пути. |
| `[!] Нет записей для экспорта` | Хранилище полностью пусто, operation экспорта отменена. |
| `[!] No such command` | Вызываемая CLI-команда не найдена в системе (проверьте через `--help`). |

## Тестирование

Для запуска тестов выполните следующие команды в вашем терминале:

```bash
pip install -r requirements.txt
pytest tests/ -v
```

Проект содержит модульные (юнит) тесты для каждого отдельного компонента программы, а также интеграционные тесты для проверки корректности взаимодействия всей системы в сборе.

## Безопасность

### Рекомендации по использованию:

1. **Мастер-пароль** должен состоять минимум из 12 символов, содержать буквы разного регистра, цифры и спецсимволы.
2. ⚠️ **Мастер-пароль невозможно восстановить!** В случае его утери все зашифрованные данные будут безвозвратно потеряны.
3. Регулярно выполняйте **экспорт хранилища** для создания резервных копий базы данных.
4. Ни в коем случае **не передавайте файл `vault.vault`** третьим лицам.
5. Никогда **не храните мастер-пароль в открытом виде** на компьютере или в текстовых файлах.
